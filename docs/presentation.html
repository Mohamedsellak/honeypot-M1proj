Bonjour. Nous sommes Mohammed et Rachid. Nous vous presentons notre projet : un honeypot. Un honeypot, c'est un piege. C'est un faux serveur. On le laisse sur le reseau pour attirer les pirates. Quand un pirate attaque, on enregistre tout ce qu'il fait.

C'est quoi un honeypot ?

Un honeypot, c'est un faux serveur.
Il fait semblant d'etre une vraie machine.
Les pirates l'attaquent... et on enregistre tout.

Image simple : c'est comme un pot de miel. 🍯
Le miel attire les insectes. Notre faux serveur attire les pirates.

Le mot honeypot veut dire pot de miel. Le miel attire les insectes. Notre faux serveur attire les pirates. Le pirate croit que c'est une vraie machine. Mais c'est un piege. On regarde tout ce qu'il fait, sans danger.

Pourquoi faire ca ? Trois raisons simples. Un : pour voir les attaques en vrai et comprendre comment les pirates travaillent. Deux : c'est sans risque, le vrai serveur n'est jamais touche. Trois : on garde les preuves. Ca nous aide a mieux nous defendre apres.

Mon piege a 3 portes

🔐
SSH

Connexion a distance. On note les mots de passe essayes.

🌐
HTTP

Un faux site web. On note les pages que le pirate cherche.

📁
FTP

Un faux partage de fichiers avec des appats.

3 services = 3 facons differentes d'attirer les pirates.

Mon piege a trois portes, trois faux services. SSH : c'est la connexion a distance. Je note tous les mots de passe que le pirate essaie. HTTP : c'est un faux site web. Je note les pages qu'il cherche, par exemple des pages sensibles. FTP : c'est un faux partage de fichiers, avec des appats comme un faux fichier secrets point txt. Plus j'ai de portes, plus j'attrape de pirates.

Tout tourne dans Docker : 6 conteneurs isoles.

Voici l'architecture de notre application, de gauche a droite. A gauche, les attaquants : on utilise Hydra et Nikto pour jouer le pirate. Ils attaquent nos trois pieges : SSH, HTTP et FTP. Chaque action devient un log signe, pour prouver qu'il n'est pas modifie. Ensuite la partie analyse : on enrichit avec le pays, on classe le type de pirate, et on stocke. Enfin, deux sorties : le tableau de bord en direct, et des fichiers d'export pour se defendre. Tout tourne dans Docker, dans six conteneurs isoles.

On reconnait le type de pirate

Le bruteforcer

Il essaie plein de mots de passe, tres vite.

Le bot

Un robot automatique qui scanne.

L'humain

Une vraie personne qui tape des commandes.

Le scanner connu

Un robot legitime (Shodan, Google...).

Le programme decide tout seul, avec des regles simples et claires.

Mon programme reconnait tout seul le type de pirate. Il y a quatre types. Le bruteforcer : il essaie beaucoup de mots de passe tres vite. Le bot : c'est un robot automatique. L'humain : une vraie personne qui tape des commandes a la main. Le scanner connu : un robot legitime, par exemple Shodan. Le programme decide avec des regles simples. Je peux toujours expliquer pourquoi.

Securite et loi

C'est securise

Le piege est enferme dans des conteneurs Docker. Le pirate ne peut pas sortir.

C'est legal

Je n'attaque personne. Je respecte le RGPD (j'anonymise les adresses).

Regle d'or : un honeypot observe, il ne riposte jamais.

Deux points importants : la securite et la loi. La securite : le piege est enferme dans des conteneurs Docker. Meme si le pirate entre, il ne peut pas sortir vers le vrai reseau. La loi : je n'attaque personne, c'est interdit. Je respecte le RGPD : j'anonymise les adresses IP. La regle d'or : un honeypot observe, il ne riposte jamais.
